# 个人作品集网站开发计划

## 1. 开发目标

基于 `PRD.md` 和 `TECH_DESIGN.md`，完成一个使用 React + TypeScript + Vite + Tailwind CSS 构建的个人作品集网站。

首版目标是交付一个简洁、高级、响应式良好的单页作品集网站，包含以下模块：

1. 首页
2. 关于我
3. 项目展示
4. 联系方式

## 2. 开发原则

- 先完成可运行版本，再逐步优化视觉与交互。
- 内容数据化，避免把个人信息和项目数据散落在组件中。
- 组件保持简单，不做过度抽象。
- 优先保证移动端体验和基础 SEO。
- 动效少而精，不影响性能。

## 3. 阶段规划

### 阶段 1：项目初始化

#### 目标

搭建基础前端工程，保证项目可以正常启动、构建和维护。

#### 任务

- 初始化 Vite + React + TypeScript 项目。
- 安装并配置 Tailwind CSS。
- 配置基础目录结构。
- 配置代码格式化和基础 lint 规则（如需要）。
- 清理默认模板内容。

#### 交付物

- 可运行的 React 项目。
- 基础目录结构：

```txt
src/
  components/
    layout/
    ui/
  sections/
  data/
  lib/
  types/
public/
  images/
```

#### 验收标准

- 本地开发服务可正常启动。
- 项目可正常构建。
- Tailwind CSS 生效。

## 4. 阶段 2：内容与数据模型

### 目标

先沉淀网站内容结构，确保页面由数据驱动。

### 任务

- 定义 `Profile`、`Project`、`ContactInfo` 类型。
- 创建个人信息数据文件。
- 创建项目展示数据文件。
- 创建联系方式数据文件。
- 准备占位图片或临时视觉素材。

### 建议文件

```txt
src/types/portfolio.ts
src/data/profile.ts
src/data/projects.ts
src/data/contact.ts
```

### 验收标准

- 页面所需核心内容都有对应数据结构。
- 后续修改文案或项目时，不需要改动主要组件结构。

## 5. 阶段 3：基础组件开发

### 目标

完成网站通用 UI 组件，为页面区块提供统一风格。

### 任务

- 开发 `Container` 组件，用于统一页面最大宽度和左右边距。
- 开发 `Button` 组件，支持主按钮和次按钮样式。
- 开发 `Card` 组件，用于项目卡片和内容卡片。
- 开发 `Badge` 组件，用于技术栈和关键词标签。
- 开发 `SectionHeading` 组件，用于统一区块标题。

### 建议文件

```txt
src/components/ui/container.tsx
src/components/ui/button.tsx
src/components/ui/card.tsx
src/components/ui/badge.tsx
src/components/ui/section-heading.tsx
```

### 验收标准

- 基础组件样式统一。
- 组件可复用。
- 组件 props 简单清晰。

## 6. 阶段 4：首页核心区块开发

### 目标

完成 4 个核心页面区块，并组合成完整单页。

### 任务

#### 6.1 首页区块

- 展示姓名或品牌名。
- 展示身份定位和一句话简介。
- 提供“查看项目”和“联系我”两个 CTA。
- 展示头像、照片或视觉图。

#### 6.2 关于我区块

- 展示个人简介。
- 展示专注领域、工作风格和核心优势。
- 使用关键词标签增强可读性。

#### 6.3 项目展示区块

- 展示 3 到 6 个精选项目。
- 每个项目展示标题、简介、封面图、技术栈、亮点和链接。
- 项目卡片在桌面端使用网格布局，移动端单列。

#### 6.4 联系方式区块

- 展示联系引导语。
- 展示邮箱和社交链接。
- 如实现表单，至少包含姓名、邮箱、项目需求和补充说明。

### 建议文件

```txt
src/sections/home.tsx
src/sections/about.tsx
src/sections/projects.tsx
src/sections/contact.tsx
src/App.tsx
```

### 验收标准

- 页面只包含 PRD 指定的 4 个核心模块。
- 结构清晰，阅读顺序自然。
- 所有核心内容均能正常显示。

## 7. 阶段 5：布局、导航与响应式

### 目标

完善页面整体布局、导航锚点和不同设备下的展示效果。

### 任务

- 开发顶部导航。
- 导航包含：首页、关于我、项目展示、联系方式。
- 点击导航项平滑滚动到对应区块。
- 开发页脚，展示版权和基础链接。
- 完成桌面端、平板端、移动端布局适配。
- 检查是否存在横向滚动。

### 建议文件

```txt
src/components/layout/header.tsx
src/components/layout/footer.tsx
```

### 验收标准

- 导航锚点可用。
- 移动端布局正常。
- CTA 按钮易点击。
- 页面无横向滚动。

## 8. 阶段 6：视觉优化与轻量动效

### 目标

让页面接近参考图的高级、简洁、强排版气质。

### 任务

- 优化 Hero 标题层级。
- 调整全局颜色、字体、间距和卡片样式。
- 增加轻微渐变、背景纹理或装饰元素。
- 增加按钮 hover、卡片 hover 反馈。
- 如引入 Framer Motion，仅用于区块渐显和轻量过渡。
- 支持 `prefers-reduced-motion`。

### 验收标准

- 页面风格统一。
- 动效不影响阅读和性能。
- 桌面端和移动端都保持良好视觉表现。

## 9. 阶段 7：SEO、可访问性与性能检查

### 目标

完成上线前基础质量检查。

### 任务

- 配置页面 title 和 meta description。
- 配置 Open Graph 和 Twitter Card。
- 为图片添加 alt。
- 检查标题层级。
- 检查表单 label。
- 检查键盘可访问性。
- 优化图片体积。
- 检查首屏加载速度。

### 验收标准

- SEO 基础信息完整。
- 主要交互可以键盘操作。
- 图片有明确 alt。
- 页面加载流畅。

## 10. 阶段 8：最终验收与部署

### 目标

完成首版交付并部署到线上。

### 任务

- 执行构建检查。
- 进行桌面端和移动端人工验收。
- 修复明显问题。
- 部署到 Vercel。
- 记录线上访问地址。

### 验收标准

- 构建通过。
- 页面内容完整。
- 链接和导航可用。
- 线上访问正常。

## 11. 开发顺序建议

推荐按以下顺序开发：

1. 初始化项目
2. 配置 Tailwind CSS
3. 定义类型和数据文件
4. 开发基础 UI 组件
5. 开发 4 个页面区块
6. 开发导航和页脚
7. 完成响应式
8. 优化视觉和动效
9. 补充 SEO 与可访问性
10. 构建检查并部署

## 12. 首版不做的内容

为控制范围，首版暂不包含：

- 博客系统
- CMS 后台
- 登录系统
- 支付功能
- 多语言
- 复杂动画
- 服务报价页
- 项目详情页

以上内容可在首版上线后按需扩展。

## 13. 风险与应对

### 13.1 内容不完整

应对：先使用占位内容完成页面结构，后续替换为真实内容。

### 13.2 视觉过度复杂

应对：优先保证清晰和高级感，减少装饰和复杂动效。

### 13.3 移动端体验不足

应对：开发过程中持续检查移动端，不等到最后统一修复。

### 13.4 项目范围膨胀

应对：严格遵守 PRD，首版只做 4 个核心模块。
