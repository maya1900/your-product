# UI 改进计划

## 1. 参考图视觉总结

参考图是一个偏杂志感、轻科技感的个人作品集页面，核心特征如下：

- 首屏大标题非常突出，文字层级强。
- 人像 / 视觉主体与标题形成强对比。
- 页面整体偏窄版长页面，移动端观感明显。
- 使用蓝色作为主要强调色。
- 区块之间有明确节奏：Hero、数据、能力卡片、作品、反馈、联系。
- 卡片设计轻量，留白较多。
- 有点状纹理、渐变、圆形装饰等视觉细节。
- 项目展示采用图文错落布局，而不是完全统一网格。
- 联系区和反馈区使用大面积蓝色块增强记忆点。

当前项目已经完成基础作品集页面，但整体还偏“标准化落地页”。下一步 UI 改进目标是：在不增加过多功能的前提下，让页面更接近参考图的高级感、个人感和作品集气质。

## 2. 改进目标

### 2.1 总体目标

让当前个人作品集页面从“基础可用”升级为“更像设计作品集”的视觉表达。

### 2.2 具体目标

1. 强化首屏视觉冲击力。
2. 增加参考图中的杂志式排版感。
3. 让项目展示更有作品集质感。
4. 强化蓝色品牌记忆点。
5. 增加适量装饰纹理，但不影响性能。
6. 保持当前 4 个核心模块不变：首页、关于我、项目展示、联系方式。

## 3. 首页 Hero 改进

### 3.1 当前问题

- Hero 标题虽然较大，但排版还偏常规。
- `Full-Stack Developer` 只是一行英文，缺少参考图那种断行和强调字的节奏。
- 视觉图是占位头像，整体记忆点不足。

### 3.2 改进方向

- 将标题改为更强的分行结构，例如：

```txt
Full-Stack
Coder &
Product Builder
```

- 对关键词使用蓝色或渐变色强调，例如 `Coder`、`Product`。
- 在标题周围加入轻微点阵、噪点或圆形线条装饰。
- Hero 视觉图采用卡片化人像区：白底卡片 + 蓝色背景块 + 圆形装饰。
- 在 Hero 下方加入简短身份标签，例如：
  - React
  - TypeScript
  - Product UI
  - Web Apps

### 3.3 实施文件

- `src/sections/home.tsx`
- `src/styles.css`
- `public/images/profile-placeholder.svg`

## 4. 关于我区块改进

### 4.1 当前问题

- 当前 About 区块比较规整，但缺少参考图里的“个人介绍卡片”感觉。
- 信息表达较平，没有特别明显的视觉锚点。

### 4.2 改进方向

- 增加一个小标题，例如 `Hi, I'm Mayang`。
- About 区域采用左右结构：左侧大号短句，右侧简介卡片。
- 将核心优势做成 2x2 小卡片，而不是普通列表。
- 关键词标签保持，但增强层级。

### 4.3 实施文件

- `src/sections/about.tsx`
- `src/components/ui/card.tsx`

## 5. 项目展示区块改进

### 5.1 当前问题

- 当前项目展示是统一三列卡片，整齐但略普通。
- 参考图的项目区更像作品陈列，有大图、小图和错落布局。

### 5.2 改进方向

- 将项目卡片改为“精选作品”布局：
  - 第一个项目占更大宽度。
  - 其他项目使用较小卡片。
  - 桌面端形成错落网格。
- 项目封面增加轻微倾斜或图层叠加效果。
- 每个项目增加更明显的项目类型标签。
- 链接按钮从纯文字改成小型 pill 按钮。

### 5.3 建议布局

```txt
Desktop:
┌───────────────┬───────────┐
│ Featured Work │ Small     │
│ Large Card    ├───────────┤
│               │ Small     │
└───────────────┴───────────┘

Mobile:
单列卡片依次排列
```

### 5.4 实施文件

- `src/sections/projects.tsx`
- `src/data/projects.ts`
- `src/components/ui/card.tsx`

## 6. 联系方式区块改进

### 6.1 当前问题

- 当前 Contact 是深色卡片，和参考图中的大面积蓝色联系区不同。
- 视觉记忆点还可以更强。

### 6.2 改进方向

- 改为蓝色主视觉联系区。
- 左侧使用大标题，例如：

```txt
Anything in Mind?
Let's Talk
```

- 右侧保留邮箱、社交链接和 CTA。
- 增加手绘线条或圆形装饰，呼应参考图的创意感。
- 邮箱按钮使用白底蓝字，增强对比。

### 6.3 实施文件

- `src/sections/contact.tsx`
- `src/components/ui/button.tsx`
- `src/styles.css`

## 7. 全局视觉改进

### 7.1 背景

当前背景已有渐变和柔光，可以进一步增加参考图里的点状纹理。

建议新增：

- `.noise-texture`
- `.dot-texture`
- `.scribble-circle`

### 7.2 色彩

建议稳定使用以下色彩：

- 主背景：`#F8FAFC`
- 主文字：`#0F172A`
- 次文字：`#64748B`
- 主蓝色：`#2563EB`
- 深蓝色：`#1D4ED8`
- 蓝紫点缀：`#7C3AED`

### 7.3 字体层级

- Hero 标题继续加大，桌面端可到 `text-8xl`。
- Section 标题保持 `text-4xl` 到 `text-5xl`。
- 项目标题使用更粗字重。

### 7.4 动效

保持轻量：

- Hero 图轻微浮动。
- 项目卡片 hover 上浮。
- 按钮 hover 位移。
- 区块首次渲染轻微渐显。

不引入复杂粒子或 Canvas。

## 8. 分阶段实施计划

### 阶段 A：Hero 强化

任务：

- 重构 Hero 标题分行。
- 增加蓝色强调文字。
- 优化头像视觉卡片。
- 增加身份标签和装饰元素。

验收：

- 首屏更接近参考图的强视觉标题。
- 移动端标题不溢出。

### 阶段 B：项目区重排

任务：

- 将三列等宽项目卡片改成 Featured Layout。
- 强化项目封面视觉。
- 优化项目链接样式。

验收：

- 桌面端项目区有错落层次。
- 移动端仍为清晰单列。

### 阶段 C：联系区品牌化

任务：

- 将深色联系卡片改为蓝色大色块。
- 增加更强 CTA 文案。
- 加入轻量装饰元素。

验收：

- 联系区成为页面结尾视觉重点。
- 邮箱和社交链接清晰可点击。

### 阶段 D：全局细节打磨

任务：

- 增加点阵 / 纹理类名。
- 统一圆角、阴影、间距。
- 检查 hover 和 reduced motion。

验收：

- 页面视觉统一。
- 无横向滚动。
- `npm run build` 通过。

## 9. 不做事项

本轮 UI 改进不做：

- 新增博客系统。
- 新增复杂 CMS。
- 新增项目详情页。
- 引入大型 UI 库。
- 引入复杂 3D、粒子、Canvas 动效。
- 修改当前四模块范围。

## 10. 验收标准

- 页面仍只包含：首页、关于我、项目展示、联系方式。
- 整体视觉更接近参考图：大标题、蓝色强调、作品集气质、创意装饰。
- 桌面端和移动端均无明显布局问题。
- 页面无横向滚动。
- 动效轻量且支持减少动态效果。
- 构建通过。

## 11. 本轮强制改进要求

> 目标：在不逐像素复制参考图的前提下，尽量贴近参考图的版式节奏、导航形态、人物图位置和作品展示方式。

### 11.1 固定导航栏

- 导航栏固定在页面顶部，不随页面滚动消失。
- 页面内容需要预留顶部空间，避免被固定导航遮挡。
- 导航栏整体保持轻量：白底、细边框、紧凑高度。

### 11.2 导航选中态与联系按钮

- 导航选中态使用“笔划 / 手绘下划线”样式，而不是传统胶囊背景。
- 导航项 hover 时也应出现类似笔划反馈。
- 右侧“联系我”按钮改为：
  - 白色背景
  - 蓝色边框
  - 蓝色文字
  - 无圆角或极小圆角

### 11.3 页面滚动节奏

- 整体页面应更紧凑，减少大段空白。
- 区块之间保持无缝滑动感，不要像传统 landing page 一样每段间隔过大。
- Hero、About、Projects、Contact 的上下 padding 需要压缩。

### 11.4 人物图片跨区布局

- 人物图片不再固定放在 Hero 右侧。
- 人物图片应作为 Hero 与 About 之间的视觉主体，形成跨区衔接。
- 图片样式参考原图：白底卡片、蓝色背景块、圆形 / 笔划装饰，突出个人形象。

### 11.5 项目展示整行交替布局

- 每个项目占一整行。
- 桌面端采用左右交替布局：
  - 第 1 个：左文字，右图片
  - 第 2 个：左图片，右文字
  - 第 3 个：左文字，右图片
- 移动端保持单列：先图片或先文字均可，但需要清晰易读。
- 项目区不再使用 Featured + 小卡片布局，而是按参考图的作品展示节奏做横向陈列。

### 11.6 本轮实施文件

- `src/components/layout/header.tsx`
- `src/sections/home.tsx`
- `src/sections/about.tsx`
- `src/sections/projects.tsx`
- `src/styles.css`

### 11.7 本轮验收标准

- 顶部导航固定可见。
- 导航选中态具备笔划感。
- 联系按钮为白底蓝框蓝字且无圆角。
- Hero 与 About 之间出现人物视觉主体。
- 项目展示为整行交替布局。
- 页面整体比当前版本更紧凑。
- `npm run build` 通过。

## 12. 视觉修正记录

### 12.1 颗粒纹理增强（已撤销）

- 该方案曾计划在 About、Projects、Contact 等不同区块加入颗粒状纹理块。
- 实际验证后视觉干扰较强，因此已移除 `.grain-patch` 及其使用位置。
- 当前保留更轻量的 `.dot-texture` 和 `.noise-texture` 作为装饰，不再额外增加颗粒纹理块。

### 12.2 Hero 内容去重

- 删除 Hero 中与 About 重复的简介文案。
- 删除 Hero 中与 About 重复的技能标签。
- Hero 保留大标题和两个主要 CTA，让首屏更接近参考图的简洁大标题结构。

### 12.3 Hero 按钮样式

- Hero 中“查看项目”和“联系我”按钮改为无圆角。
- “联系我”按钮改为白底、蓝色边框、蓝色文字。

### 12.4 人物图跨区层级修复

- Hero 人物图向下跨到 About 区域。
- Hero 区和人物图增加更高层级，About 区降低层级，避免人物图被 About 白色背景遮挡。
- About 区增加顶部空间，用来承接跨区人物图。

### 12.5 Contact 可读性修复

- Contact 区必须保持蓝色 CTA 色块正常显示。
- 若通用 Card 默认背景覆盖 Contact 背景，需要使用明确优先级确保蓝色背景生效。
- Contact 标题、描述、社交链接应使用白色或高对比颜色，不能出现淡到不可读的情况。
