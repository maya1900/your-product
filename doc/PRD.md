# 个人作品集网站 PRD

## 1. 项目目标

本项目要做一个简洁、高级、可信的个人作品集网站，用于展示个人身份、代表项目，并引导访客联系你。

## 2. 页面范围

页面只保留 4 个核心模块：

1. 首页
2. 关于我
3. 项目展示
4. 联系方式

> 采用单页结构，通过导航锚点跳转。

## 3. 首页

### 3.1 目标

首页是第一印象，需要快速说明你是谁、擅长什么、适合什么合作。

### 3.2 内容

- 姓名或品牌名
- 身份定位
- 一句话简介
- 主要 CTA：查看项目
- 次要 CTA：联系我
- 头像、照片或视觉图

### 3.3 要求

- 标题要足够大，形成视觉焦点。
- 信息要少而清晰。
- 保持高级感和留白。

## 4. 关于我

### 4.1 目标

让访客快速了解你的背景、专注方向和工作方式。

### 4.2 内容

- 个人简介
- 专注领域
- 工作风格
- 核心优势

### 4.3 展示方式

- 一段简短文案
- 若干关键词标签

## 5. 项目展示

### 5.1 目标

用真实项目证明能力。

### 5.2 内容

- 项目名称
- 简介
- 封面图
- 技术栈
- 项目亮点或结果
- Demo / GitHub / 案例链接（如有）

### 5.3 要求

- 优先展示 3 到 6 个精选项目。
- 每个项目要有清晰层级。
- 后续可扩展项目详情页。

## 6. 联系方式

### 6.1 目标

提供最直接的联系入口。

### 6.2 内容

- 联系引导语
- 邮箱
- 社交链接
- 表单或外链入口

### 6.3 如实现表单

建议字段：

- 姓名
- 邮箱
- 项目需求
- 补充说明

## 7. 视觉风格

- 白色或浅灰背景
- 深色文字
- 单一强调色系，建议蓝色或蓝紫色
- 大留白
- 卡片化布局
- 克制的阴影和渐变

## 8. 交互要求

- 页面加载轻微淡入
- 卡片 hover 有反馈
- 按钮状态清晰
- 尊重减少动态效果设置

## 9. 响应式要求

- 桌面端可用双栏或网格
- 平板端减少列数
- 移动端单列展示
- 不允许横向滚动

## 10. SEO 与可访问性

必须支持：

- 页面标题
- meta description
- Open Graph
- Twitter Card
- 语义化 HTML
- 图片 alt

必须满足：

- 标题层级清晰
- 键盘可操作
- 表单有 label
- 对比度足够

## 11. 验收标准

- 页面只包含首页、关于我、项目展示、联系方式。
- 首页足够清晰、有吸引力。
- 项目内容可通过数据文件维护。
- 桌面端和移动端都正常。
- SEO 基础完整。
- 整体风格统一。
